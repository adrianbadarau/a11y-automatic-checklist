# Agent Rules

The following rules must be strictly adhered to by any Antigravity agent operating in this repository:

1. **No Commits**: Do not ever commit anything. Never run `git commit`.
2. **Test-Driven Development (TDD)**: Always use a TDD approach when developing. Write failing tests first, ensure they fail, then write the minimal code required to pass the tests, and finally refactor if necessary.
3. **Document each new feature** in by adding a new file in the docs folder. The file should be named after the feature and should be written in a way that is easy to understand for someone who is not familiar with the codebase. Make sure to add a link to the new file in the README.md file.
4. **Always create a new branch** for each new feature. The branch name should be in the format `feat/feature-name`.
